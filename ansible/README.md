# Ansible Configuration

Ansible playbooks and roles for configuring all services in the homelab. Includes deployment orchestration, service configuration, and comprehensive health validation.

## 📋 Quick Reference

```bash
# Deploy everything (with automatic validation)
ansible-playbook playbooks/site.yml

# Deploy runners by dedicated scope
ansible-playbook playbooks/github-runners.yml -e "runner_inventory_group=github_runners_org" -e "runner_vars_file=github_runners.yml"
ansible-playbook playbooks/github-runners.yml -e "runner_inventory_group=github_runners_personal_homelab" -e "runner_vars_file=github_runners_personal_homelab.yml"
ansible-playbook playbooks/github-runners.yml -e "runner_inventory_group=github_runners_personal_litbot" -e "runner_vars_file=github_runners_personal_litbot.yml"

# Configure MinIO backend container for Terraform remote state
ansible-playbook playbooks/minio-tfstate.yml

# Validate components (health checks)
ansible-playbook playbooks/validate.yml

# Deploy just monitoring stack
ansible-playbook playbooks/monitoring.yml

# Validate single host
ansible-playbook playbooks/validate.yml --limit=api
```

## 📁 Directory Structure

### `playbooks/`
Master playbooks for orchestration and configuration.

| Playbook | Purpose | Hosts | Dependencies |
|----------|---------|-------|--------------|
| `site.yml` | Master orchestration | all | None (includes all others) |
| `monitoring.yml` | Prometheus + Grafana + exporters | monitoring, all, mysql_servers | None |
| `validate.yml` | Health checks (standalone) | all | None |
| `php-server.yml` | Apache + PHP installation | api, gapi, homepage, site | None |
| `mysql-server.yml` | MySQL installation + setup | db | None |
| `api.yml` | API service configuration | api | php-server |
| `gapi.yml` | Game API service config | gapi | php-server |
| `homepage.yml` | Homepage service config | homepage | None |

**Execution Flow (site.yml):**
```
base → php-server → mysql-server → minio-tfstate → github-runner(org) → github-runner(homelab) → github-runner(litbot) → api → gapi → homepage → monitoring → validate
```

Each playbook can run independently for targeted deployments.

### `roles/`
Reusable configuration bundles.

```
roles/
└── base/
    └── main.yml          # Base system setup (SSH, users, firewall, packages)
```

The `base` role is imported by `site.yml` as the first step and handles foundational system configuration.

### `inventory/`
Host and variable definitions.

```
inventory/
├── hosts.yml            # Dynamic inventory (AUTO-GENERATED by Terraform)
│                        # DO NOT EDIT - regenerated on terraform apply
│                        # Contains: proxmox_nodes, api_servers, mysql_servers, monitoring groups
│
├── secrets.yml          # Vault-encrypted secrets
│                        # Contains: vault_mysql_prometheus_password, runner_ssh_private_key, api keys, etc.
│                        # Edit: ansible-vault edit inventory/secrets.yml
│
├── homepage_data.yml    # Static configuration for homepage service
│                        # Contains: service definitions, feature flags
│
└── group_vars/
    └── all.yml          # Global variables for all hosts
                         # Contains: NTP, package versions, common paths
```

**Important:** `hosts.yml` is auto-generated by Terraform (`ansible.tf`) and reflects current infrastructure state. Do not manually edit it.

### `templates/`
Jinja2 configuration templates that render with variables.

**Monitoring Stack:**
- `prometheus.yml.j2` - Prometheus config (targets, scrape intervals, alerts)
- `prometheus-alerts.yml.j2` - Alert rule definitions
- `prometheus.service.j2` - Prometheus systemd service
- `grafana.service.j2` - Grafana systemd service
- `grafana.env.j2` - Grafana environment variables
- `grafana-datasources.yml.j2` - Auto-provision Prometheus datasource
- `grafana-dashboards.yml.j2` - Auto-provision dashboard config
- `homelab-overview-dashboard.json.j2` - Main dashboard definition
- `node-exporter.service.j2` - Node Exporter systemd service
- `mysql-exporter.service.j2` - MySQL Exporter systemd service
- `mysql-exporter.cnf.j2` - MySQL Exporter credentials config

**Service Configurations:**
- `apache-api.conf.j2` - API service Apache vhost
- `apache-gapi.conf.j2` - GAPI service Apache vhost
- `homepage-services.yaml.j2` - Homepage service definitions

### `files/`
Static files copied to hosts as-is.

```
files/
├── api_deploy_key       # SSH key for API service deployment
├── gapi_deploy_key      # SSH key for GAPI service deployment
└── sql_imports/         # Database initialization SQL files
    ├── marctowl_gapi_*.sql      (20+ GAPI database tables)
    └── marctowl_itslitapi_*.sql (20+ ItsLit API database tables)
```

### Configuration Files

- `ansible.cfg` - Ansible runtime configuration
  - Host key checking disabled (for new containers)
  - Inventory path: `inventory/hosts.yml`
  - Vault password handling configured

- `requirements.yml` - Ansible Galaxy dependencies
  - Currently empty (no external roles used)
  - Add community collections if needed

## 🚀 Common Tasks

### Full Deployment

```bash
# Deploy everything with automatic validation
ansible-playbook playbooks/site.yml

# With verbose output (troubleshooting)
ansible-playbook playbooks/site.yml -v

# Run specific tags only
ansible-playbook playbooks/site.yml --tags=monitoring

# Dry-run (check mode)
ansible-playbook playbooks/site.yml --check
```

### Targeted Deployments

```bash
# Deploy only monitoring stack
ansible-playbook playbooks/monitoring.yml

# Deploy only to specific host
ansible-playbook playbooks/site.yml --limit=api

# Deploy only database
ansible-playbook playbooks/mysql-server.yml

# Focus on exporters (all hosts)
ansible-playbook playbooks/monitoring.yml --limit=all
```

### Health Validation

```bash
# Full health check (all components)
ansible-playbook playbooks/validate.yml

# Check single host
ansible-playbook playbooks/validate.yml --limit=api

# Quick connectivity test
ansible all -i inventory/hosts.yml -m ping

# Verbose validation output
ansible-playbook playbooks/validate.yml -v
```

### Troubleshooting

```bash
# Check current inventory
ansible-inventory -i inventory/hosts.yml --list

# Verify variables for a host
ansible-inventory -i inventory/hosts.yml --host=api | jq .

# Test connectivity to all hosts
ansible all -i inventory/hosts.yml -m ping

# Run shell command on host group
ansible api_servers -i inventory/hosts.yml -m shell -a "systemctl status node_exporter"

# View secrets (with vault password)
ansible-vault view inventory/secrets.yml
```

## 🔐 Secrets Management

### Vault File

Secret values are stored in `inventory/secrets.yml` (encrypted with Ansible vault):

```bash
# View encrypted secrets
ansible-vault view inventory/secrets.yml

# Edit secrets
ansible-vault edit inventory/secrets.yml

# Encrypt a file
ansible-vault encrypt filename

# Decrypt a file (one-time)
ansible-vault decrypt filename
```

**Required Secrets:**
- `vault_mysql_prometheus_password` - MySQL Exporter user password
- `runner_ssh_private_key` - SSH private key installed on GitHub runner hosts for runner-initiated Ansible SSH access
- Any API keys or deploy credentials for services

### Using Vault in Playbooks

```yaml
- name: Load vault secrets
  include_vars:
    file: ../inventory/secrets.yml
  no_log: yes  # Don't display in output
```

### Running with Vault

```bash
# Provide password at prompt
ansible-playbook playbooks/site.yml --ask-vault-pass

# Use password file
ansible-playbook playbooks/site.yml --vault-password-file=~/.vault_password

# Or set environment variable
export ANSIBLE_VAULT_PASSWORD_FILE=~/.vault_password
ansible-playbook playbooks/site.yml
```

## 📊 Monitoring & Validation

### Automatic Validation (During Deployment)

When `playbooks/site.yml` runs, handlers automatically validate:

```
Task: Enable and start Prometheus
  → Handler: validate prometheus health
     → Check: HTTP GET http://localhost:9090/-/healthy
     → Retries: 5 (waits up to 10 seconds)

Task: Enable and start Grafana
  → Handler: validate grafana health
     → Check: HTTP GET http://localhost:3000/api/health

Task: Enable and start node_exporter (all hosts)
  → Handler: validate node exporter
     → Check: Port 9100 connectivity

Task: Enable and start mysql_exporter (db only)
  → Handler: validate mysql exporter
     → Check: Port 9104 connectivity
```

**If validation fails:** Playbook stops immediately with clear error message.

### Standalone Validation

`playbooks/validate.yml` performs comprehensive health checks:

**Prometheus Validation:**
- Health endpoint check
- Count active targets
- List dropped targets
- Fail if zero targets active

**Grafana Validation:**
- Health endpoint check
- List provisioned datasources
- Verify Prometheus datasource

**Exporter Validation (All Hosts):**
- Node Exporter: Check port 9100 + metrics endpoint
- MySQL Exporter: Check port 9104 + database connectivity
- Per-host status reporting

## 🛠️ Customization

### Add New Service Stack

```bash
# 1. Create playbook
touch playbooks/myservice.yml

# 2. Add to site.yml
echo "- name: Configure MyService" >> playbooks/site.yml
echo "  import_playbook: myservice.yml" >> playbooks/site.yml

# 3. Create templates as needed
mkdir -p templates/myservice.conf.j2

# 4. Test syntax
ansible-playbook --syntax-check playbooks/myservice.yml

# 5. Deploy
ansible-playbook playbooks/site.yml
```

### Modify Monitoring Targets

Edit `templates/prometheus.yml.j2` to add/remove scrape jobs:

```yaml
scrape_configs:
  - job_name: "custom-service"
    static_configs:
      - targets: ["10.0.1.100:9100"]
```

Then redeploy:
```bash
ansible-playbook playbooks/monitoring.yml
```

### Add Vault Secrets

```bash
# Edit vault file
ansible-vault edit inventory/secrets.yml

# Add new variable (YAML format)
my_new_secret: "secret_value"

# Use in playbook
- name: Use secret
  debug:
    msg: "{{ my_new_secret }}"
  no_log: yes
```

## 🐛 Troubleshooting Common Issues

### "Failed to connect" to host

**Cause:** Host not reachable or SSH keys not configured
```bash
# Verify host is in inventory
ansible-inventory -i inventory/hosts.yml --list

# Test connectivity
ansible all -i inventory/hosts.yml -m ping

# Check SSH connections manually
ssh -vvv user@host
```

### "module_setup failed" on first run

**Cause:** Common on fresh containers, usually resolves on retry
```bash
# Run again (Python may need installation)
ansible-playbook playbooks/site.yml

# Or force Python discovery
ansible all -m setup
```

### Vault password not working

**Cause:** Wrong password or file not found
```bash
# Test vault file integrity
ansible-vault view inventory/secrets.yml

# If locked out, run with prompt
ansible-playbook playbooks/site.yml --ask-vault-pass
```

### Handler validation fails (during deploy)

**Cause:** Service took too long to start or port not accessible
```bash
# Increase retries in monitoring.yml (line ~243)
retries: 10  # Try 10 times instead of 5

# Or run validation separately after manual restart
ansible-playbook playbooks/validate.yml
```
